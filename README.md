# MADLAD Arabic-to-French Fine-Tuning Project

## Project Overview

This project fine-tunes the `google/madlad400-10b-mt` model for Arabic-to-French translation specialized in Islamic literature. The pipeline creates a doubled dataset (with and without Arabic diacritics) and properly splits train/test data.

---

## Prerequisites

Install required Python packages:

```bash
pip install pandas openpyxl pyarabic numpy torch transformers datasets unbabel-comet sentence-transformers scikit-learn tqdm
```

**Notes:** 
- The preprocessing script will download the MADLAD tokenizer (~500MB) on first run to validate token lengths
- The test extraction script will download a multilingual sentence transformer model (~420MB) for GPU-accelerated deduplication
- **GPU highly recommended** for test extraction (100x faster than CPU)

## Execution Workflow

### Step 1: Data Preprocessing
**Script:** `create_combined_dataset.py`

**Input:** `data_french.xlsx` (raw Arabic-French parallel data)

**Output:** `combined_dataset.xlsx` (doubled dataset)

**What it does:**
- Assigns `original_id` to each row for duplicate tracking
- Loads MADLAD tokenizer for token length validation
- Creates VERSION A: Arabic WITH tashkeel (diacritics)
- Creates VERSION B: Arabic WITHOUT tashkeel
- Cleans Arabic source text (removes brackets, special symbols, HTTP links)
- Cleans French target text (preserves accents: é, è, ê, à, ç, ù, etc.)
- Filters out invalid rows (HTTP links, insufficient text)
- **Filters out entries exceeding 1024 tokens** (MADLAD model limit)
- Adds version tracking (`with_tashkeel` / `without_tashkeel`)
- Combines both versions and shuffles
- **Tracks duplicates using `original_id` column**

**Run:**
```bash
python create_combined_dataset.py
```

---

### Step 2: Test Set Extraction
**Script:** `extract_test_set.py`

**Input:** `combined_dataset.xlsx`

**Output:** 
- `training_dataset.xlsx` (remaining data after test extraction)
- `test_dataset.xlsx` (exactly 200 samples)

**What it does:**
- **Phase 1: Exact Duplicate Removal**
  - Randomly samples 200 unique `original_id` values (seed=42)
  - Extracts **ALL versions** of sampled IDs for test set (~400 rows total)
  - Removes **ALL versions** of sampled IDs from training data
  - Verifies no `original_id` overlap
  
- **Phase 2: GPU-Accelerated Semantic Deduplication** 
  - Loads multilingual sentence transformer model (paraphrase-multilingual-mpnet-base-v2)
  - Computes embeddings for all texts **on GPU** (100x faster than CPU string matching)
  - Calculates cosine similarity matrices between training and test
  - Removes training examples with >95% similarity on **both source AND target**
  - Catches semantic duplicates: typos, punctuation, word order, paraphrases
  
- **Result:** Zero data leakage, comprehensive deduplication, GPU-accelerated
- Saves both sets separately with tracking columns

**Run:**
```bash
python extract_test_set.py
```

---

### Step 3: Convert to JSONL Format
**Script:** `to_json.py`

**Input:** 
- `training_dataset.xlsx`
- `test_dataset.xlsx`

**Output:**
- `translations.jsonl` (training data)
- `test.jsonl` (validation data)

**What it does:**
- Converts Excel files to JSONL format
- Removes tracking columns (`original_id`, `version`) - not needed for training
- Preserves UTF-8 encoding for Arabic and French characters
- Format: `{"source_text": "...", "target_text": "..."}`

**Run:**
```bash
python to_json.py
```

---

### Step 4: Fine-Tune MADLAD Model
**Script:** `MADLAD.py`

**Input:**
- `translations.jsonl` (training)
- `test.jsonl` (validation)

**Output:** `madlad400-10b-finetuned-ar-to-fr-islamic/` (model checkpoints)

**What it does:**
- Loads google/madlad400-10b-mt (10 billion parameters)
- Formats input with `<2fra>` prefix (French language code)
- Uses COMET metric for evaluation
- Training settings optimized for 10B model:
  - BFloat16 precision
  - Batch size: 1
  - Gradient accumulation: 32 steps
  - 3 epochs
  - Learning rate: 2e-5

**Run:**
```bash
python MADLAD.py
```

⚠️ **WARNING:** This requires significant GPU resources (multiple high-end GPUs recommended)

## Critical: Duplicate Handling in Doubled Dataset

**Why This Matters:**
Since the dataset is doubled (with/without Arabic diacritics), each original example appears twice. Additionally, there may be near-duplicates with slight variations. Without proper handling, test examples could leak into training data.

**Two-Phase Deduplication Strategy:**

### Phase 1: Exact Duplicate Removal (`original_id` tracking)
1. **Preprocessing:** Each original row gets a unique `original_id` 
2. **Test Extraction:** 
   - Sample 200 unique `original_id` values
   - Extract **ALL rows** with those IDs (both versions) → ~400 test rows
   - Remove **ALL rows** with those IDs from training

### Phase 2: GPU-Accelerated Semantic Deduplication (>95% similarity)
3. **Embedding-Based Matching:**
   - Load multilingual sentence transformer model (supports Arabic & French)
   - Compute semantic embeddings for all texts **on GPU** (100x faster)
   - Calculate cosine similarity matrices (vectorized, very fast)
   - Remove training examples where BOTH source ≥95% AND target ≥95% similar
   - Catches semantic duplicates: typos, punctuation, word order, paraphrases

**Example:**
```
Phase 1 - original_id tracking:
  Original row 42: "مرحبا" → "Bonjour"
  After doubling:
    - Row with original_id=42, version=with_tashkeel: "مَرْحَبًا" → "Bonjour"
    - Row with original_id=42, version=without_tashkeel: "مرحبا" → "Bonjour"
  
  If original_id=42 sampled for testing:
    ✓ BOTH versions → test set
    ✓ BOTH versions removed from training

Phase 2 - GPU semantic matching:
  Test embedding: "مرحبا بك" → [0.23, -0.45, 0.67, ...] (768-dim vector)
  Train embedding: "مرحبا بك!" → [0.24, -0.44, 0.68, ...] (768-dim vector)
  Cosine similarity: source=0.98, target=0.97
  Result: ✓ Removed (both >0.95 threshold, semantically identical)
```

**Why Both Phases:**
- Phase 1 catches exact duplicates from dataset doubling
- Phase 2 catches near-duplicates that Phase 1 might miss
- Together: Comprehensive, zero-leakage split

## Key Configuration Details

### Language Codes
- **Source:** Arabic (ara)
- **Target:** French (fra) - ISO 639-3 code
- **MADLAD prefix:** `<2fra>` (automatically added to inputs)

### Data Processing
- **Arabic preprocessing:** With and without tashkeel (diacritics)
- **French preprocessing:** Preserves all accents and special characters
- **Token length filtering:** Automatically removes entries exceeding 1024 tokens
- **Duplicate tracking:** Uses `original_id` to track pairs across versions
- **Test set size:** 200 unique examples (~400 total rows with both versions)
- **No data leakage:** Test examples (all versions) fully excluded from training

### Model Settings
- **Model:** google/madlad400-10b-mt
- **Max sequence length:** 1024 tokens
- **Evaluation metric:** COMET score (Unbabel/wmt22-comet-da)
- **Training device:** Auto-distributed across available GPUs

2. **Test Extraction:**

- This is a one-time operation that ensures data quality.

## Troubleshooting

### Missing data_french.xlsx
Make sure your input file `data_french.xlsx` has these columns:
- `source_text` (Arabic text)
- `target_text` (French text)

### Out of Memory during training
If you encounter OOM errors:
1. Reduce `per_device_train_batch_size` in MADLAD.py (try 1 or lower if possible)
2. Increase `gradient_accumulation_steps` to maintain effective batch size
3. Reduce `max_length` (line 94) if your texts are shorter

### COMET model download issues
The COMET model will be downloaded automatically on first run. Ensure you have:
- Stable internet connection
- Sufficient disk space (~2GB for COMET model)

### Too many entries filtered during preprocessing
If the token length filter removes too many entries:
- Check your input data - entries might be very long paragraphs
- The 1024 token limit is a hard constraint from the MADLAD model
- Consider splitting very long texts into separate entries before preprocessing
- Check the preprocessing output to see how many were filtered

### Fuzzy matching removes too many/too few training examples
Adjust the `SIMILARITY_THRESHOLD` in `extract_test_set.py` (line 112):
- **Too many removed** (training set too small): Lower threshold (try 0.90 or 0.92)
- **Too few removed** (still see duplicates): Raise threshold (try 0.97 or 0.98)
- **Default 0.95**: Good balance for most datasets (95% cosine similarity)
- Edit line 112: `SIMILARITY_THRESHOLD = 0.95` → change the number (0.0 to 1.0 scale)

### GPU not detected / Using CPU instead
If the script uses CPU instead of GPU:
- Verify CUDA is installed: Run `python -c "import torch; print(torch.cuda.is_available())"`
- Should print `True` if GPU is available
- Install CUDA-enabled PyTorch: Visit https://pytorch.org/get-started/locally/
- Check GPU memory: Sentence transformer needs ~2GB VRAM
- The script will automatically fall back to CPU if GPU is unavailable

### Out of GPU memory during deduplication
If you get CUDA out of memory errors:
- Reduce batch size in line 27: `batch_size=32` → try `batch_size=16` or `batch_size=8`
- Close other GPU applications
- The model needs ~2GB VRAM; embeddings need additional memory based on dataset size
